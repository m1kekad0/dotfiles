# 全体

IMPORTANT: 以下のルールはすべての会話・タスクに常時適用されます。必ず遵守してください。

- **すべてのメッセージを日本語で出力すること**（ファイル名・メソッド名・コマンド等の固有名詞は除く）
- ツール実行中のステータスメッセージ・思考過程・進捗報告も日本語で出力すること
- 英語で応答してはいけません

# ソースコード実装

- doc コメント（Javadoc, JSDoc 等）は **private を含むすべてのクラス・メンバ** に必ず記載すること
- 本文中のインラインコメントも、可読性を損なわない範囲で充実させること
- **コメントはすべて日本語**で記述すること

# GitHub PR 操作

- PR を編集する前に必ず `gh pr list --state all` で open 状態を確認すること
- open 状態の PR のみ編集してよい。merged / closed の PR は編集しないこと
- 新規コミットが存在し対象 PR が open でない場合は、新規 PR を作成すること

# 技術ブログ作成依頼

- ブログ作成を依頼された場合は、**過去の記事を参照して体裁を合わせること**
  - 過去記事が参照できない場合は、記事リンクの提示をユーザーに求めること
- 本文には適切な段落構成を設けること
- 記事本文とは別に、以下を必ず提示すること：
  - タイトル
  - slug
  - タグ
  - サマリ（200字程度）
